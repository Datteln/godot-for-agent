# 🧠 一、Claude Code 的缓存核心思想（你现在 vs 它）

你当前系统：

> ❌ token 阈值 + system boundary + 单点 cache_control

Claude Code 风格：

> ✅ **“Prefix-based deterministic caching + 多断点 + 内容稳定性哈希 + 分层不可变前缀”**

---

## 🔥 核心差异（非常重要）

| 维度          | 你当前实现              | Claude Code 风格               |
| ----------- | ------------------ | ---------------------------- |
| 缓存触发        | token ≥ 1024       | prefix 结构稳定即可                |
| cache 单位    | 单 message boundary | **prefix segment（多段）**       |
| cache key   | 隐式由模型推断            | **显式 content-addressed key** |
| 变化检测        | 没有                 | **stable fingerprint（关键）**   |
| tool schema | 参与前缀但不版本化          | **强版本锁定**                    |
| RAG context | 当 system 拼进去       | **独立 segment cache**         |
| cache 控制    | 单点 marker          | **multi-breakpoint caching** |

---

# 🚀 二、升级目标（对标 Claude Code）

把当前系统升级为：

> 🧩 **Claude-style Prefix Caching System for Coding Agents**

核心能力：

### 🎯 目标

* Prefix caching 命中率 ↑↑↑
* repo-level / tool-level cache reuse
* 多轮 agent 调用成本下降 50~80%
* 支持 IDE / coding agent / MCP 工具链

# 🧱 三、架构升级设计（重点）

---

# 3.1 Prompt 结构重构（Claude 风格核心）

将 message 拆成 **4 层不可变结构**

```text
┌──────────────────────────────┐
│  L0: SYSTEM_CORE              │  ← 永久稳定（人格/规则）
├──────────────────────────────┤
│  L1: TOOL_SCHEMA             │  ← 版本化（函数/MCP）
├──────────────────────────────┤
│  L2: PROJECT_CONTEXT         │  ← repo summary / docs
├──────────────────────────────┤
│  L3: EPHEMERAL_CONTEXT       │  ← 当前文件 / diff / RAG
├──────────────────────────────┤
│  L4: CONVERSATION            │  ← user / assistant
└──────────────────────────────┘
```

---

# 3.2 Claude-style Cache Key（核心新增）

## ❌ 删除：token threshold 触发

---

## ✅ 新增：内容地址缓存键（Cache Fingerprint）

